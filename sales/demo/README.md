# Demo Materials Directory

## File Naming Convention

### Video Files (Not Stored in Repo)
- **Master**: `ODAVL_Demo_90s_YYYY-MM-DD_master.mp4` (Full quality, local storage)
- **Web**: `ODAVL_Demo_90s_YYYY-MM-DD_web.mp4` (Compressed for web, <50MB)
- **Social**: `ODAVL_Demo_30s_YYYY-MM-DD.mp4` (Short clips for social media)

### Supporting Materials (Stored in Repo)
- `DEMO_90S.md` - Complete script with timing and commands
- `RECORDING_CHECKLIST.md` - Production quality checklist
- `README.md` - This file, naming conventions and workflow

## Production Workflow

1. **Preparation**: Follow `RECORDING_CHECKLIST.md` completely
2. **Recording**: Execute `DEMO_90S.md` script with stopwatch
3. **Export**: Create both master and web versions using naming convention
4. **Storage**: Master files in secure local/cloud storage, NOT in Git repo
5. **Distribution**: Upload web version to YouTube, website hosting

## Distribution Locations

### Primary Platforms
- **Website**: Embedded on odavl.studio homepage hero section
- **YouTube**: SEO-optimized listing with description and tags
- **Sales Materials**: Linked in email templates and pitch presentations

### Social Media Versions
- **LinkedIn**: 60-second version for professional audience
- **Twitter**: 30-second highlight reel with key value props
- **Demo Calls**: Direct link for prospects requesting demonstration

## File Size Guidelines

- **Master**: No limit (archive quality)
- **Web**: <50MB for fast loading
- **Social**: <25MB for platform compatibility
- **Thumbnail**: 1920x1080 JPG, <500KB

## Version Control

- **Scripts**: Version controlled in Git (this directory)
- **Video Files**: External storage with version dates in filename
- **Updates**: New versions increment date, keep old versions archived

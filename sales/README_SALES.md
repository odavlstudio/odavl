## üéØ How to Use This Kit

### Customizing the One-Pager

1. Open `ONEPAGER.md` in your preferred Markdown editor
2. Replace placeholder contact information:
   - `[YOUR_NAME]` ‚Üí Your full name
   - `[YOUR_EMAIL]` ‚Üí Your business email
   - `[YOUR_PHONE]` ‚Üí Your phone number
3. Update company-specific metrics in the results section
4. Add your headshot or company logo if desired

### Exporting to PDF

**Option A - Print to PDF (Recommended):**

1. Open `ONEPAGER.md` in VS Code with Markdown Preview
2. Use Ctrl+Shift+P ‚Üí "Markdown: Open Preview"
3. Print Preview ‚Üí Save as PDF
4. Ensure margins are set to "Minimum" for best layout

**Option B - Browser Method:**

1. Copy content to GitHub Gist or GitLab snippet
2. Use browser's "Print to PDF" function
3. Select "More settings" ‚Üí "Background graphics" for best results

### Presenting the Pitch Deck

**Browser-Based Presentation:**

1. Open `PITCH_DECK.md` in GitHub/GitLab for clean rendering
2. Use presenter view or full-screen mode
3. Navigate slides by scrolling or using slide breaks

**PDF Export for PowerPoint:**

1. Follow same PDF export process as one-pager
2. Import PDF slides into PowerPoint for customization
3. Each H1 heading represents a new slide break

### Running the 5-Minute Demo

**Prerequisites:**

- VS Code with ODAVL Doctor extension installed
- Sample repository with ESLint warnings (use fallback repo if needed)
- Timer or stopwatch for precise timing

**Demo Execution:**

1. Follow exact timing in `DEMO_5MIN.md`
2. Practice the script 2-3 times before live presentation
3. Have fallback commands ready if primary demo encounters issues
4. Keep terminal/VS Code windows sized for screen sharing visibility

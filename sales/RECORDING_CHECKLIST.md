# ODAVL Demo Recording Checklist

## Pre-Recording Environment Setup

### System Requirements

- [ ] **Screen Resolution**: 1920x1080 (16:9 ratio for optimal platform compatibility)
- [ ] **Audio**: Headset/microphone tested and levels checked
- [ ] **Lighting**: Consistent, no glare on screen
- [ ] **Recording Software**: OBS Studio/ScreenFlow/Camtasia configured
- [ ] **Frame Rate**: 30fps minimum for smooth playback

### Desktop Preparation

- [ ] **Close All Unnecessary Apps**: Slack, email, notifications disabled
- [ ] **Clean Desktop**: Remove personal files, shortcuts
- [ ] **Browser**: Fresh session, clear cache, no personal bookmarks visible
- [ ] **VS Code**: Clean workspace, no personal extensions visible
- [ ] **Terminal**: Clear history, clean prompt

### ODAVL Project Setup

- [ ] **Repository**: Fresh clone or clean working directory
- [ ] **Dependencies**: All packages installed (`pnpm install`)
- [ ] **Test Files**: Sample code with 20+ ESLint warnings prepared
- [ ] **ODAVL CLI**: Confirmed working (`pnpm odavl:observe`)
- [ ] **Extension**: Installed and verified (`code --list-extensions | grep odavl`)

## Recording Settings

### Audio Configuration

- [ ] **Input Level**: -12dB to -6dB (avoid clipping)
- [ ] **Background Noise**: Minimal, use noise gate if needed
- [ ] **Test Recording**: 10-second sample to verify clarity
- [ ] **Backup Audio**: Secondary recording method (phone app) as safety

### Video Configuration

- [ ] **Resolution**: 1920x1080 native (no scaling)
- [ ] **Bitrate**: 5000-8000 kbps for crisp code visibility
- [ ] **Color Profile**: sRGB for consistent display across platforms
- [ ] **Mouse Cursor**: Visible, highlighting enabled

## Content Preparation

### Script Readiness

- [ ] **Demo Script**: `sales/DEMO_90S.md` reviewed and memorized
- [ ] **Timing**: Practice runs completed, 90-second target achieved
- [ ] **Commands**: All terminal commands tested and working
- [ ] **Outputs**: Expected JSON responses verified

### Backup Scenarios

- [ ] **Retake Plan**: Ready to restart if errors occur
- [ ] **Network Issues**: Local demo environment (no external dependencies)
- [ ] **Command Failures**: Backup commands prepared
- [ ] **Timing Issues**: Shorter/longer versions of key sections prepared

## Recording Session

### Pre-Roll (Before Recording)

- [ ] **Final Audio Check**: Levels and clarity confirmed
- [ ] **Screen Clean**: Desktop organized, irrelevant windows closed
- [ ] **Posture/Energy**: Professional presentation mindset
- [ ] **Stopwatch**: Timer ready to monitor 90-second target

### During Recording

- [ ] **Pace**: Speak clearly, not rushed (demo should feel natural at 90 seconds)
- [ ] **Mouse Movement**: Deliberate, professional cursor movement
- [ ] **Command Typing**: Clear, visible typing (not too fast)
- [ ] **Transitions**: Smooth between browser, VS Code, terminal
- [ ] **Error Recovery**: If mistake occurs, complete take and note for editing

### Post-Recording Immediate Check

- [ ] **Duration**: Verify recording is 85-95 seconds (editing buffer)
- [ ] **Audio Sync**: No audio/video drift
- [ ] **Visual Quality**: Code is readable, no compression artifacts
- [ ] **Content Complete**: All key messages delivered
- [ ] **CTA Clear**: odavl.studio URL visible and mentioned

## File Management

### Export Settings

- [ ] **Format**: MP4 (H.264, AAC audio)
- [ ] **Quality**: High quality, but under 50MB for web distribution
- [ ] **Filename**: `ODAVL_Demo_90s_YYYY-MM-DD.mp4`
- [ ] **Thumbnail**: Extract frame showing clean ODAVL interface

### Backup & Storage

- [ ] **Master File**: Full quality version saved locally
- [ ] **Web Version**: Compressed version for website/social media
- [ ] **Upload Locations**: YouTube, Vimeo, website hosting prepared
- [ ] **Backup Copy**: Stored in secure cloud location

## Contingency Plans

### Common Issues & Solutions

- **Audio Problems**: Have backup recording device ready
- **Screen Recording Fails**: Alternative recording software installed
- **Command Errors**: Practice environment with known-good commands
- **Timing Issues**: Flexible script sections for natural pacing
- **Technical Difficulties**: Simple screen-sharing demo as fallback

### Retake Scenarios

- **Major Errors**: Complete retake (better than editing around mistakes)
- **Minor Issues**: Note timestamp for post-production editing
- **Performance Issues**: Energy/enthusiasm inconsistent with brand

**Final Reminder**: This demo is the first impression for many prospects. Professional quality and clear value demonstration are essential for launch success.

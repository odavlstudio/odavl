## Calendar Booking Instructions

### Scheduling Best Practices

1. **Time Zones:** Always specify time zone clearly
2. **Buffer Time:** Schedule 5-10 min buffer between demo calls
3. **Confirmation:** Send confirmation email 24h before meeting
4. **Backup Plans:** Always have backup demo environment ready
5. **Recording:** Ask permission, provide recording link within 4h

### Meeting Preparation Checklist

**For Demo Calls:**

- [ ] Demo repository tested and working
- [ ] Backup demo environment verified
- [ ] Prospect research completed (tech stack, recent news)
- [ ] Custom talking points prepared
- [ ] Pilot program pricing confirmed

**For Pilot Kickoffs:**

- [ ] Repository access confirmed
- [ ] Installation requirements verified
- [ ] Safety configurations prepared
- [ ] Success metrics framework ready
- [ ] Emergency contact information shared

**For Results Reviews:**

- [ ] Comprehensive metrics report generated
- [ ] Team feedback collected and analyzed
- [ ] Comparison charts prepared (before/after)
- [ ] Next phase recommendations ready
- [ ] Commercial terms prepared (if pilot successful)

### Follow-up Automation

**Within 1 Hour:**

- Send meeting summary email
- Share relevant resources/links
- Schedule next meeting if agreed upon

**Within 24 Hours:**

- Provide promised materials (demos, reports, proposals)
- Connect on LinkedIn if relationship building
- Update CRM/tracking system with meeting notes

**Within 48 Hours:**

- Follow up on any action items
- Send additional resources if requested
- Confirm next steps or schedule follow-up

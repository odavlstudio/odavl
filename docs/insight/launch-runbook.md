## Environment Setup

### 1. Production Environment Variables

Create `.env.production` in `apps/cloud-console/`:

```bash
# ============================================================================
# ODAVL Cloud Console - Production Environment Variables
# ============================================================================

# Application
NODE_ENV=production
NEXT_PUBLIC_APP_URL=https://cloud.odavl.studio
NEXT_PUBLIC_API_URL=https://api.odavl.studio

# Database (PostgreSQL)
DATABASE_URL=postgresql://username:password@host:5432/odavl_production?sslmode=require

# NextAuth.js
NEXTAUTH_URL=https://cloud.odavl.studio
NEXTAUTH_SECRET=<generate-with-openssl-rand-base64-32>

# OAuth Providers
GITHUB_ID=<github-oauth-app-client-id>
GITHUB_SECRET=<github-oauth-app-client-secret>
GOOGLE_CLIENT_ID=<google-oauth-client-id>
GOOGLE_CLIENT_SECRET=<google-oauth-client-secret>

# Stripe
STRIPE_SECRET_KEY=sk_live_<your-stripe-secret-key>
STRIPE_PUBLISHABLE_KEY=pk_live_<your-stripe-publishable-key>
STRIPE_WEBHOOK_SECRET=whsec_<your-webhook-secret>

# Stripe Product IDs (create these in Stripe dashboard)
STRIPE_PRICE_PRO_MONTHLY=price_<pro-monthly-id>
STRIPE_PRICE_PRO_ANNUAL=price_<pro-annual-id>
STRIPE_PRICE_TEAM_MONTHLY=price_<team-monthly-id>
STRIPE_PRICE_TEAM_ANNUAL=price_<team-annual-id>

# Telemetry (PostHog)
NEXT_PUBLIC_POSTHOG_KEY=<posthog-project-key>
NEXT_PUBLIC_POSTHOG_HOST=https://app.posthog.com

# Sentry (Error Monitoring)
SENTRY_DSN=https://<key>@<org>.ingest.sentry.io/<project>
SENTRY_AUTH_TOKEN=<sentry-auth-token>

# Email (SendGrid)
SENDGRID_API_KEY=SG.<your-sendgrid-api-key>
SENDGRID_FROM_EMAIL=noreply@odavl.studio

# Redis (for rate limiting, caching)
REDIS_URL=redis://:password@host:6379

# Feature Flags
NEXT_PUBLIC_ENABLE_CLOUD_ANALYSIS=true
NEXT_PUBLIC_ENABLE_TEAM_COLLABORATION=true
NEXT_PUBLIC_ENABLE_TELEMETRY=true

# Rate Limiting
RATE_LIMIT_MAX_REQUESTS=100
RATE_LIMIT_WINDOW_MS=60000

# Logging
LOG_LEVEL=info
LOG_FORMAT=json
```

## Cloud Deployment

### Option 1: Vercel (Recommended)

**1. Install Vercel CLI:**
```bash
npm install -g vercel
```

**2. Link project:**
```bash
cd apps/cloud-console
vercel link
# ‚úÖ Select: ODAVL Studio
# ‚úÖ Project: cloud-console
```

**3. Set environment variables:**
```bash
# Set all production env vars
vercel env add DATABASE_URL production
vercel env add NEXTAUTH_SECRET production
vercel env add STRIPE_SECRET_KEY production
# ... (repeat for all vars)

# Or use Vercel dashboard:
# https://vercel.com/odavl-studio/cloud-console/settings/environment-variables
```

**4. Deploy to production:**
```bash
# Deploy with production environment
vercel --prod

# Example output:
# üîç Inspect: https://vercel.com/odavl-studio/cloud-console/...
# ‚úÖ Production: https://cloud.odavl.studio
```

**5. Verify deployment:**
```bash
# Test health endpoint
curl https://cloud.odavl.studio/api/health

# Expected: {"status":"ok","version":"1.0.0"}
```

### Option 2: Docker + AWS/GCP

**1. Build Docker image:**
```bash
cd apps/cloud-console

# Build production image
docker build -t odavl-cloud:1.0.0 -f Dockerfile.prod .

# Test locally
docker run -p 3000:3000 \
  --env-file .env.production \
  odavl-cloud:1.0.0
```

**2. Push to container registry:**
```bash
# AWS ECR
aws ecr get-login-password --region us-east-1 | docker login --username AWS --password-stdin <account>.dkr.ecr.us-east-1.amazonaws.com
docker tag odavl-cloud:1.0.0 <account>.dkr.ecr.us-east-1.amazonaws.com/odavl-cloud:1.0.0
docker push <account>.dkr.ecr.us-east-1.amazonaws.com/odavl-cloud:1.0.0

# Or Google Container Registry
docker tag odavl-cloud:1.0.0 gcr.io/<project-id>/odavl-cloud:1.0.0
docker push gcr.io/<project-id>/odavl-cloud:1.0.0
```

**3. Deploy to ECS/Cloud Run:**
```bash
# AWS ECS
aws ecs update-service --cluster odavl-prod --service cloud-console --force-new-deployment

# Or Google Cloud Run
gcloud run deploy cloud-console \
  --image gcr.io/<project-id>/odavl-cloud:1.0.0 \
  --platform managed \
  --region us-central1
```

## Rollback Procedures

### Emergency Rollback (If Launch Fails)

**Scenario 1: Critical Bug in Cloud Console**

```bash
# Option 1: Revert Vercel deployment
vercel rollback https://cloud.odavl.studio

# Option 2: Revert to previous commit
git revert <commit-hash>
git push origin main
vercel --prod

# Rollback time: <5 minutes
```

**Scenario 2: Database Migration Issue**

```bash
# Restore from backup
psql $DATABASE_URL < backup-20251212-143000.sql

# Verify restoration
psql $DATABASE_URL -c "SELECT COUNT(*) FROM users;"

# Rollback time: <10 minutes
```

**Scenario 3: Stripe Webhook Issues**

```bash
# Disable webhook temporarily
# 1. Go to Stripe dashboard ‚Üí Webhooks
# 2. Disable endpoint
# 3. Fix issue
# 4. Re-enable endpoint

# Manual subscription sync if needed
pnpm run stripe:sync-subscriptions
```

**Scenario 4: NPM Package Broken**

```bash
# Unpublish broken version (within 72 hours)
npm unpublish @odavl/cli@1.0.0

# Or deprecate
npm deprecate @odavl/cli@1.0.0 "Critical bug, use 1.0.1"

# Publish fix
npm version patch
npm publish
```

**Scenario 5: VS Code Extension Broken**

```bash
# Unpublish from marketplace (contact support)
# Or publish hotfix immediately
vsce publish 1.0.1

# Users auto-update within 24 hours
```

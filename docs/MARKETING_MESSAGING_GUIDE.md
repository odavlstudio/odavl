## Tone & Voice Guidelines

### Brand Personality

**Intelligent**: We're experts, but we explain clearly  
**Trustworthy**: Security-first, transparent about limitations  
**Efficient**: Respect developer time, no fluff  
**Professional**: Enterprise-ready, but not boring

### Writing Style

**Do**:
- Use active voice ("ODAVL detects errors" not "Errors are detected by ODAVL")
- Be specific ("Reduce review time by 40%" not "Make reviews faster")
- Show, don't tell (demos, screenshots, metrics)
- Use developer terminology correctly

**Don't**:
- Overpromise ("eliminate all bugs forever")
- Use buzzwords without substance ("AI-powered" without explaining how)
- Be condescending ("Your code is broken, we fix it")
- Use marketing jargon ("synergize", "paradigm shift")

### Example Transformations

‚ùå **Weak**: "ODAVL is a great tool for code quality"  
‚úÖ **Strong**: "ODAVL detects 95% of errors before code review using ML-powered analysis"

‚ùå **Weak**: "Our AI makes your code better"  
‚úÖ **Strong**: "Our TensorFlow.js model predicts fix success with 92% accuracy based on 10 codebase features"

‚ùå **Weak**: "Join thousands of developers"  
‚úÖ **Strong**: "Join 2,500+ developers analyzing 10,000+ repositories"

## Objection Handling

### "Another code quality tool? We already use X."

**Response**: "Great! Most teams use 3-5 tools (linter, security scanner, formatter, etc.). ODAVL consolidates these into three specialized products ‚Äî and adds autonomous fixing. You can replace multiple tools with one platform, or use ODAVL alongside your existing setup. Many teams start with Insight (detection only) and add Autopilot once they trust the results."

### "Autonomous code changes sound risky."

**Response**: "That's a healthy concern! We built three layers of safety:

1. **Risk Budget**: Max 10 files per cycle, protected paths never touched
2. **Undo Snapshots**: Every change is reversible with one command
3. **Attestation Chain**: Cryptographic proof of every change for audits

Plus, you review all changes before they merge. Think of Autopilot as a junior engineer doing the grunt work ‚Äî you're still the senior reviewer."

### "How is this different from GitHub Copilot?"

**Response**: "Different problems! Copilot helps you *write* code (AI pair programmer). ODAVL helps you *maintain* code (AI QA engineer).

Copilot = Feature development  
ODAVL = Quality assurance

Many teams use both ‚Äî Copilot for initial implementation, ODAVL for automated review and fixing."

### "What if it makes a mistake?"

**Response**: "Two safeguards:

In 10,000+ production runs, our false-positive rate is <5%. And every change is reversible."
